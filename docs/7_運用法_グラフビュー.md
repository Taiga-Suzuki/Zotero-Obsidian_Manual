# Obsidian運用法 グラフビューについて

## 概説

グラフビューでは、各mdファイルの参照関係(リンク)を可視化することができる。  
リンクを密に設定している場合、グラフビューが煩雑になるので、適宜「グラフビューに反映されない参照法」を使用することが肝要である。

## リンクの種類と仕様について

### 通常リンク (紫字, dataview外)

[[mdファイル名]] の形で記述  
グラフビューに反映される  
command を押しながらリンクにカーソルを合わせることで、リンク先のプレビューが可能。GA画像を設定していると便利。  
command + 左クリックで、open as new tab が可能。

### 外部リンク (緑字)

[表示名] (URL) の形で記述  
(括弧間のスペースは不要)  
グラフビューには反映されないので、関係はあるが(このページから飛びたいが)グラフビュー的には近づけたくないものを記述するのに最適  
リンク先のプレビューは不可(obsidianのmdファイルであっても)  
command + 左クリック を押しても、同じタブで開いてしまう。  

URLにあたる部分は、外部URLでも、obsidian URLでも良い。  
obsidian URLは、下記から取得できる。  
ページ右上の「...」 > パスをコピー > Obsidian URL として

### Dataview内のリンク (紫字)

Dataviewにより自動的に生成
グラフビューには反映されない。
commandを押さなくても、リンク先のプレビューが可能。

#### 運用上のtips

Dataview内のリンクがグラフビューには反映されないことを利用することで、グラフビューの煩雑性を抑えつつ、
さまざまな視点でのポータルページを作成できる(様々なポータルページから、1つの論文ページに飛べる)。

逆に、ポータルページと論文ページとのリンクを作成したい場合、手動で通常リンクを記載する必要がある。  
運用上、下記のようにした方が良い

ポータルページ:  
　論文ページへの通常リンクは使わない。Dataviewリンクのみ。
 
論文ページ:  
　複数のポータルページへのリンクを設定する。但し、通常リンクは1(~2)個に抑えて、残りは緑色のリンクにする。  
　論文ページ間のリンクは、余程近いものでない限りは、緑色のリンクを用いる。  

このようにすると、グラフビューが単純化でき、新たなポータルができた際に、グラフビューにおける線の付け替えがしやすい。
